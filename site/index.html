    Infrastructure

    1.  Disaster Recovery Plan: Ensure HA for critical network services and to provide a clear path to data restoration for stateful services in the event of hardware failure or data corruption.
    2.  IP Address Management: Local network IPAM, VPN and VLAN information.
    3.  MAC Address Tables: Tables containing devices on the LAN and their hardware MAC addresses.
    4.  Network Security Policy: This infrastructure operates on the Principle of Least Privilege. No device or service is granted more network access than is strictly required for its primary function. Security is maintained through physical isolation, logical segmentation, and encrypted transit.
    5.  Logical Map: Mermaid.js flowchart focusing on servers and services.
    6.  Physical Map: Mermaid.js flowchart focusing on physical connections.

    Services

    1.  Beszel Agent: Agent for Beszel Hub, a server utilization monitor.
    2.  Beszel Hub: A self-hosted server utilization monitor.
    3.  Caddy: Lightweight, open-source Web server written in Go. Used as a reverse-proxy for creating unique domains for locally hosted services.
    4.  Cloudflared: A secure reverse-proxy tunnel for hosting private services. Currently, Immich and Uptime Kuma are utilizing this service.
    5.  CUPS Print Server: Print server for USB-connected Brother_HL-L2300D_series laser printer.
    6.  Dockge: A fancy, easy-to-use and reactive self-hosted Docker compose.yaml stack-oriented manager.
    7.  F1 Replay Timing: Visualization of real-time track data and telemetry synced to F1 live and replays.
    8.  Fluidd: A Web UI for managing the 3D-printer.
    9.  Glance: A server dashboard Web UI.
    10.  Glances: A top / htop alternative for the Web browser.
    11.  Home Assistant: A home automation and IoT dashboard.
    12.  Homebox: An inventory and organization system built for the home user.
    13.  Immich: Photo gallery and backup service.
    14.  IT-Tools: Handy tools for network administrators and developers.
    15.  Moonraker: An API for the Klipper firmware.
    16.  Nebula-Sync: Synchronize settings between Pi-hole instances.
    17.  Nextcloud: Self-hosted cloud storage and collaboration platform.
    18.  Network File System (NFS): Remote file system access.
    19.  Nginx (Network Portfolio): The world's most popular Web Server, high performance Load Balancer, Reverse Proxy, API Gateway and Content Cache.
    20.  ntopng: Network traffic probe.
    21.  Obsidian LiveSync: A CouchDB database for synchronizing Obsidian Vaults.
    22.  OpenSpeedTest: An HTML5 network speed test server.
    23.  Pi-hole: A DNS server and ad/malware blocker.
    24.  Portainer: A Docker container management platform.
    25.  Portainer Agent: An agent for connecting remote servers to Portainer.
    26.  Portracker: A self-hosted, real-time port monitoring and discovery tool.
    27.  Remote Desktop Protocol (RDP): Remote desktop access over the local network. (not exposed to the internet)
    28.  Server Message Block (SMB): Remote file system access.
    29.  Spoolman: Manage 3D-printer filament inventory.
    30.  Secure Shell (SSH): Remote shell access to servers
    31.  Syncthing: Open decentralized file synchronization.
    32.  Technitium: An open source authoritative as well as recursive DNS server that can be used for self hosting a DNS server for privacy & security.
    33.  ttydBridge: Access the host terminal in a Web browser.
    34.  Uptime Kuma: A self-hosted service monitoring tool.
    35.  WireGuard Server: A lightweight, secure remote access VPN server.
    36.  yt-dlp WebUI: A Web UI and RPC server for yt-dlp.

    My Aliases

    Update PCRs Script

    In this GitHub repository I have written a Bash script, update-pcrs, that automates the process of clearing and registering new PCRs and regenerating the initramfs after a firmware or kernel upgrade. The script is full-featured with flags for using custom PCRs (defaults to 0+4+7+11), usage help, checking the version, and defining the device path.

    Update PCRs

    1.  Defining Terminal Aliases: How to define Bash terminal aliases in their own file to avoid a cluttered .bashrc file. The methodology is different on Debian and RHEL / Fedora based Linux distributions.

    3.  Setup SSH Login Email Notification: How to set up an email notification delivered to your inbox every time a new SSH session is established; utilizing a Bash script, msmtp and pam_exec.so.
