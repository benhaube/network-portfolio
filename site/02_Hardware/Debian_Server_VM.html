    Debian Server VM

    Debian  Debian Docs

    Overview

    • Role: Primary DNS server (Technitium Cluster)
    • Hostname(s): debian-vm
    • Hypervisor: ZimaOS (KVM/QEMU)
    • Operating System:
      •  Debian Linux 13 (Trixie)
    • Host Node:
      •  ZimaBoard 2 NAS
    • Credentials:
      •  Bitwarden SSH key: "server-admin@debian-vm"

    Resource Allocation

    • vCPUs: 2 Cores
    • RAM: 2 GB
    • Storage: 50GB Virtual Disk Image
      • Disk Location on Host: /media/nvme0n1p1/VM/debian-vm/cd175b11

    Network Configuration

    Network Interface:

    Interface IP Address MAC Address Notes
    enp1s0 192.168.50.6 52:54:00:28:BB:EA Bridged Adapter (Appears as separate device on LAN)

    IP Configuration:

    VLAN Hostname(s) DNS Servers Gateway
    vlan50 debian-vm.internal 127.0.0.1 192.168.50.1

    Services / Docker Containers

    Native Linux:

    Status Service Port(s) Role / Notes
    Active SSH (Secure Shell) 22 Remote terminal access.
    Active Technitium 53 443 5380 An open source authoritative as well as recursive DNS server that can be used for self hosting a DNS server for privacy & security.

    Docker:

    Status Service Port(s) Role / Notes
    Active Beszel Agent 45876 Agent for Beszel Hub (hosted on Raspberry Pi 4B Server).
    Active Dockge 5001 A fancy, easy-to-use and reactive self-hosted Docker compose.yaml stack-oriented manager.

    Maintenance & Snapshots

    Critical Configurations
    1. Do not modify NTP settings or enable systemd-timesyncd. It has been replaced with chrony.

    2. The VM is configured to boot automatically when the host server boots.

      • If a ZimaOS update breaks this configuration it can be re-applied with the following command on the host machine:

        1
        sudo virsh autostart <vm_name>

      • To disable the 'autostart' feature: (not recommended)

        1
        sudo virsh autostart <vm_name> --disable

    3. This server has email notifications configured for new SSH sessions and unattended-upgrades. See SSH Login Email Notification for documentation.

    Update Policy:

    • sudo apt update && sudo apt upgrade (Standard Debian)
    • unattended-upgrades enabled for critical bug and security fixes.

    Snapshot Policy:

    • Perform external snapshots before major changes.

    Warning!

    1. The kvm/qemu hypervisor on ZimaOS only supports external snapshots.

    2. To create an external snapshot, run the following command as root: 

      1
      virsh snapshot-create-as <vm_name> <snapshot-name> "External snapshot description" --disk-only --atomic

    Recovery:

    • If the VM is corrupted there are external snapshots to roll back to.
    • Also, on the main storage pool on the ZimaBoard 2 NAS in the /media/Quick_Storage/Backup/virsh-backups directory there is a backup disk image and XML settings file.
